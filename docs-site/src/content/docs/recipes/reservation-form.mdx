This recipe adds an end-to-end reservation flow: guests submit a request through
[`@carte/views`](/plugins/carte-views/) and
[`@carte/reservations`](/plugins/carte-reservations/) creates a pending
reservation, sends confirmation email in `ctx.waitUntil`, and confirms the
booking through a tokenized route.

## 1. Configure reservation settings

In EmDash settings for `@carte/reservations`, set the capacity and email values
used by your restaurant:

```text
defaultPartySize: 2
maxPartySize: 8
holdTtlSeconds: 600
hostEmail: host@example.test
fromEmail: reservations@example.test
```

Expected output:

```text
Settings saved for @carte/reservations
```

The plugin declares `content:read`, `content:write`, and `email:send`. Capacity
holds expire after ten minutes and capacity updates are atomic.

## 2. Add the form page

Create `src/pages/reserve.astro`:

```astro
---
import { ReservationForm } from "@carte/views";
---

<ReservationForm
  action="/_emdash/api/plugins/carte-reservations/submit"
  defaultDate="2026-06-12"
  defaultTime="19:00"
  partySizeOptions={[1, 2, 3, 4, 5, 6, 7, 8]}
/>
```

Expected output from the Astro dev server:

```text
13:00:00 [200] /reserve
```

## 3. Submit a reservation

```bash
curl -s -X POST http://localhost:8787/_emdash/api/plugins/carte-reservations/submit \
  -H 'content-type: application/json' \
  -d '{
    "guestName": "Ada Lovelace",
    "email": "ada@example.test",
    "date": "2026-06-12",
    "time": "19:00",
    "partySize": 2,
    "notes": "Window table if available"
  }'
```

Expected output:

```json
{
  "ok": true,
  "status": "pending",
  "token": "..."
}
```

The route creates a pending reservation, decrements capacity, and queues host
and guest email work with `ctx.waitUntil`.

## 4. Confirm the token round-trip

Open the confirmation link sent to the guest, or use the token returned by your
test submission:

```bash
curl -s 'http://localhost:8787/_emdash/api/plugins/carte-reservations/confirm?token=PASTE_TOKEN_HERE'
```

Expected output:

```json
{
  "ok": true,
  "status": "confirmed"
}
```

## 5. Cancel by token when needed

```bash
curl -s 'http://localhost:8787/_emdash/api/plugins/carte-reservations/cancel-by-token?token=PASTE_TOKEN_HERE'
```

Expected output:

```json
{
  "ok": true,
  "status": "cancelled"
}
```

Cancellation restores capacity and queues cancellation email outside the
response path.
