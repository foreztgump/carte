This recipe adds an end-to-end reservation flow: guests submit a request through
[`@carte/views`](/plugins/carte-views/) and
[`@carte/reservations`](/plugins/carte-reservations/) creates a pending
reservation, settles notification work in-request through the sandbox adapter,
and confirms the booking through a tokenized route.

## 1. Configure reservation settings

In EmDash settings for `@carte/reservations`, set the token secret and the
default seat capacity used by your restaurant:

```text
tokenSecret: paste-a-long-random-secret     (secret)
capacityPerSlot: 24
```

Expected output:

```text
Settings saved for @carte/reservations
```

The plugin declares `content:read`, `content:write`, and `email:send`. Capacity
holds expire after ten minutes and are claimed through the storage-backed
capacity path.

## 2. Add the form page

Create `src/pages/reserve.astro`:

```astro
---
import { ReservationForm } from "@carte/views";
---

<ReservationForm
  action="/_emdash/api/plugins/carte-reservations/submit"
  defaultDate="2026-06-12"
  defaultTime="19:00"
  partySizeOptions={[1, 2, 3, 4, 5, 6, 7, 8]}
/>
```

Expected output from the Astro dev server:

```text
13:00:00 [200] /reserve
```

## 3. Submit a reservation

```bash
curl -s -X POST http://localhost:4321/_emdash/api/plugins/carte-reservations/submit \
  -H 'content-type: application/json' \
  -d '{
    "guestName": "Ada Lovelace",
    "guestEmail": "ada@example.test",
    "date": "2026-06-12",
    "slot": "19:00",
    "partySize": 2
  }'
```

Expected output:

```json
{
  "ok": true,
  "status": 200,
  "reservationId": "...",
  "confirmationToken": "..."
}
```

The route creates a pending reservation, claims capacity, and settles the guest
received email before the sandboxed request completes.

## 4. Confirm the token round-trip

Open the confirmation link sent to the guest, or use the `confirmationToken`
returned by your test submission:

```bash
curl -s 'http://localhost:4321/_emdash/api/plugins/carte-reservations/confirm?token=PASTE_TOKEN_HERE'
```

Expected output:

```json
{
  "ok": true,
  "status": 200,
  "reservationId": "..."
}
```

## 5. Cancel by token when needed

```bash
curl -s 'http://localhost:4321/_emdash/api/plugins/carte-reservations/cancel-by-token?token=PASTE_TOKEN_HERE'
```

Expected output:

```json
{
  "ok": true,
  "status": 200,
  "reservationId": "..."
}
```

Cancellation restores capacity and settles the cancellation email during the
request.
