This recipe connects a storefront checkout form to the EmDash 0.18 Carte
runtime: [`@carte/core`](/plugins/carte-core/) owns menu data,
[`@carte/reservations`](/plugins/carte-reservations/) owns table holds, and
[`@carte/orders-backend`](/plugins/carte-orders-backend/) creates a
Tender-hosted checkout. [`@carte/views`](/plugins/carte-views/) renders the
cart. Tender and its payment provider own card entry, so Carte infrastructure
never receives raw PAN, CVC, or expiry data.

## 1. Confirm the sandboxed Carte runtime

The verified harness registers the three sandboxed Carte plugins in
`sandboxed: []` and uses the workerd sandbox runner:

```js
emdash({
  sandboxed: [carteCore, carteReservations, carteOrdersBackend],
  sandboxRunner: "@emdash-cms/sandbox-workerd",
});
```

Expected output from the harness dev server:

```text
Loaded sandboxed plugin carte-core:0.1.0
Loaded sandboxed plugin carte-reservations:0.1.0
Loaded sandboxed plugin carte-orders-backend:0.1.0
```

Use the same route prefix the harness mounts:
`/_emdash/api/plugins/<plugin-id>/<route>`.

## 2. Configure Tender checkout settings

Install and configure the Tender payment model described in `MIGRATION.md`:
`@carte/orders-backend` plus Tender's core and Stripe provider. Store Stripe
keys only in the Tender Stripe provider settings. In
`@carte/orders-backend`, save the settings that the checkout route reads:

```text
tenderBaseUrl: https://tender.example
tenderPluginToken: tender_plugin_token...     (secret)
currency: usd
```

Expected output:

```text
Settings saved for @carte/orders-backend
Secret values stored by EmDash
```

The backend creates Tender hosted-checkout charges through `@tender/sdk`. It no
longer stores Stripe secret keys or a Stripe webhook secret in Carte settings.

## 3. Render a checkout page

Create `src/pages/checkout.astro` in the host Astro site:

```astro
---
import { OrderingCart, OrderingCheckout } from "@carte/views";

const cart = {
  id: "cart_demo_001",
  currency: "USD",
  lineItems: [
    {
      id: "line_001",
      menuItemId: "squash-risotto",
      name: "Squash Risotto",
      quantity: 2,
      unitPrice: 2400,
      modifiers: [{ id: "extra-sage", name: "Extra sage", price: 100 }],
    },
  ],
  totals: { subtotal: 5000, tax: 400, serviceFee: 0, tip: 0, total: 5400 },
};
---

<OrderingCart cart={cart} />
<OrderingCheckout
  action="/_emdash/api/plugins/carte-orders-backend/checkout"
  cart={cart}
/>
```

Expected output from the Astro dev server:

```text
13:00:00 [200] /checkout
```

## 4. Create a Tender checkout

```bash
curl -s -X POST http://localhost:4321/_emdash/api/plugins/carte-orders-backend/checkout \
  -H 'content-type: application/json' \
  -d '{
    "cartId": "cart_demo_001",
    "successUrl": "https://restaurant.example/success",
    "cancelUrl": "https://restaurant.example/checkout",
    "customerEmail": "guest@example.test",
    "lineItems": [
      {
        "name": "Squash Risotto",
        "quantity": 2,
        "unitAmount": 2500
      }
    ]
  }'
```

Expected output:

```json
{
  "checkoutUrl": "https://tender.example/checkout/..."
}
```

Redirect the guest to `checkoutUrl`. Do not collect card fields in Carte forms;
Tender's payment provider handles card number, CVC, expiry, SCA, and wallet
payment UI.

## 5. Payment state after checkout

Do not forward payment-provider webhooks to Carte. The EmDash 0.18
`@carte/orders-backend` plugin does not expose a Carte Stripe webhook route.
Tender's Stripe provider owns that webhook, and `@carte/orders-backend` keeps
only the idempotent Tender transaction seam (`applyTenderTransaction`) for
future payment-state delivery.

The checkout route itself completes in-request: it stores a short-lived cart
hold, calls Tender, and returns the hosted checkout URL. Sandboxed Carte routes
do not rely on a post-response primitive.
