## Rationale

This feature treats `docs/competitive-analysis/adoptable-patterns.md` as the source of truth for backlog creation: only MUST-ADOPT and SHOULD-ADOPT patterns become Linear sub-tasks, and each task preserves the exact plugin/file:line evidence plus the proposed Carte target. OQ annotations summarize only the findings that directly influence modifier complexity, photo handling, notification/cancellation scope, or tax strategy so Milestone 1 can resolve those questions without re-reading every teardown.
